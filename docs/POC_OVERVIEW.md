# Game Coaching Platform PoC Definition

## 1. 概要 (Overview)
本プロジェクトは、ゲーマーおよびeSports市場をターゲットとした、真にユーザーファーストなコーチングプラットフォームの構築を目指すものである。
既存の国内プラットフォームにおける機能不足や信頼性の課題を解決し、Discordとの密な連携を通じて、コーチ・生徒双方にとってストレスのない体験を提供する。

## 2. 長期的な構想 (Long-term Vision)
最終的に目指すプラットフォームの状態は以下の通りである。

### コーチ (Coach)
- **ブランディングと販売**: 自身のスキルや実績を魅力的に提示し、コーチングプランを自由に作成・販売できる。
- **シームレスな誘導**: 生徒を自身のDiscordサーバーへスムーズに誘導し、コミュニティ形成を促進できる。
- **スケジュール管理**: 日程調整の自動化とリマインド機能により、管理コストを最小化し、「うっかり忘れ」を防ぐ。
- **売上管理**: ダッシュボードで収益を可視化・管理できる。

### 生徒 (Student)
- **コーチ検索**: 自分に合ったコーチを直感的に探せる。
- **ストレスフリーな利用**: 決済から日程調整までをワンストップで行え、DMでの煩雑なやり取りを不要にする。
- **安心感**: 信頼できる決済システムとリマインド機能により、安心してサービスを利用できる。

## 3. PoCの範囲 (PoC Scope)
今回のProof of Concept (PoC) では、**「決済 (Payment)」** 機能の実装にフォーカスする。
最小限の機能セット (MVP) で構築し、ペルソナは「開発者自身」と「そのコーチ」の2名に限定して検証を行う。

### PoCの目的
- Stripe Connectを用いた決済フローの技術検証
- アプリケーション経由での売上管理の有用性確認
- (将来的な) 日程調整機能実装への足掛かり

### 対象外 (Out of Scope for PoC)
- 自動日程調整機能 (今回は口頭/手動)
- Discord Bot連携 (将来実装)
- 一般ユーザー向けの公開・登録機能

## 4. ユーザーストーリー (User Stories)

### 現状の課題 (As-Is)
1. **依頼・調整**: X (旧Twitter) のDMでコーチング依頼と日程調整を行っているため、履歴が分散し管理が煩雑。
2. **決済**: PayPay等の個人間送金を利用しており、法的な懸念や経理上の不透明さがある。
3. **実施**: リマインドがないため、予定を忘れてコーチングが流れてしまうことがある。

### PoCでの解決策 (To-Be)
1. **依頼・調整**: 日程調整自体はまだ口頭で行うが、契約のトリガーをアプリ上に集約する。
2. **決済**: アプリケーション上から Stripe Connect を利用して安全に決済を行う。
3. **売上管理**: コーチは管理画面から自身の売上を確認・出金管理ができる。

## 5. 機能要件 (Functional Requirements)
- **ユーザー認証**: 最小限のログイン機能 (コーチ/生徒)。
- **決済機能**: Stripe Connect を用いた決済処理。
- **出金サイクル**: 月1回 (Stripe Connect Expressの手数料削減のため)。
- **ダッシュボード**: コーチ向けの売上確認画面。
